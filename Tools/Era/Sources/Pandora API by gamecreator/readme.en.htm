Pandora's API

version 2.0.1 (c) gamecreator 2014-2015

The purpose of Pandora's API is to allow easy addition of user-defined ERM receivers.
SDK is located in Mods\Pandora's API\SDK.
Pandora's API implements following functionality:

  • new Era event "OnPandora'sApiReady" to indicate when user can call Pandora's API's functions;
    • 

  • SDK namespace ErmVar to provide access to all ERM variables;
    • 

  • functions:

    • int __stdcall RegisterErmReceiver(char name[2], ErmInnerHandler handler, int type)
Adds user-defined receiver to ERM. Type determines the number of receiver arguments and must be one of the following:
0 - no arguments (UN-like);
1 - one argument (BM-like);
2 - one to five arguments (OB-like);
3 - one argument (FU-like);
4 - four arguments (DO-like);
5 - two arguments (HT-like).
      • 

    • int __stdcall GenericErmHandler(ErmHandler handler, char c, int num, void* todo, void* mes)
Helper function. Parses all commands into simple representation and calls user-defined handler for each.
      • 

    • void ConnectPandorasAPI()
SDK function. Loads Pandora's API for user plugin.
      • 

    • void ApplyArg(ErmArgument& arg, int& value)
SDK helper function. Applies integer command argument to an integer variable depending on how it was passed to the receiver (set/check/get).
      • 

    • bool CheckArgs(ErmCommand& cmd, ErmArgumentFormat format)
SDK helper fuction. Checks if received arguments satisfy specified requirements.
      • 

    • bool IsValidVar(char vartype, int number)
SDK helper function. Checks if an ERM variable of specified type and number exists.
    
This mod includes two receivers:

    • PA - can be used to set up Pandora's Boxes and Events (readme)
      • 

    • QU - can be used to set up Seer Huts and Quest Guards (readme)
    
Their sources are located in Mods\Pandora's API\Sources.
